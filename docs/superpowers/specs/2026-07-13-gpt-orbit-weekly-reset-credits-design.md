# Gpt Orbit Weekly 与重置次数徽标设计

**日期：** 2026-07-13

**状态：** 已批准

**范围：** 在保留现有 Gpt Orbit 的前提下，增加一个可独立安装、只显示周额度与可用重置次数的 Windows 11 桌面悬浮组件

**前置规格：** 继续遵守 `2026-07-12-gpt-orbit-floating-widget-design.md` 中关于透明悬浮窗、拖动、置顶、托盘、位置恢复、隐私、缓存和故障恢复的要求；本规格只覆盖周额度专用版本的差异

## 1. 产品目标

现有 **Gpt Orbit** 保持不变，继续提供 5 小时额度正面与周额度翻面。

新增独立产品 **Gpt Orbit Weekly**。它只显示 Codex 周额度，不在任何可见文本、无障碍名称或交互状态中呈现 5 小时额度。组件右侧常驻一个红色圆形玻璃徽标，显示 OpenAI 赠送且当前仍可使用的额度重置次数。

标准版和周版必须可以同时安装、同时运行、分别记忆位置，互不覆盖应用数据和安装记录。

## 2. 构建与应用隔离

两个产品共享同一套 React、Rust、额度服务、托盘、窗口控制和测试代码。周版通过 Tauri 配置叠加与 Vite 构建模式产生，不复制整个工程。

周版固定身份：

- 产品名：`Gpt Orbit Weekly`
- 主程序名：`gpt-orbit-weekly`
- 应用标识符：`com.codex-orbit.weekly`
- 前端构建变量：`VITE_ORBIT_VARIANT=weekly`

Tauri 使用 JSON Merge Patch 配置叠加。周版配置覆盖产品名、主程序名、应用标识符、周版前端构建命令，并完整替换 `app.windows` 数组以写入周版初始几何。由于数组不会逐项合并，周版窗口条目必须逐字保留标准版的透明、无边框、不可调整、无阴影、默认置顶、跳过任务栏和启动隐藏等安全属性，只修改标题与几何；不能因配置叠加而丢失任何窗口安全设置。

应用标识符不同，因此 WebView 数据、配置目录、窗口位置和安装记录天然隔离。托盘标题与提示从运行时产品配置读取，不能硬编码为标准版名称。

## 3. 周额度专用交互

周版只有 `collapsed` 与 `expandedWeekly` 两个稳定状态，不存在前后面翻转。

### 3.1 默认状态

- 周额度主体仍是精确的 **148 × 148 px** 圆形紫蓝玻璃球。
- 仅显示 `WEEK LEFT`、周剩余百分比和周重置倒计时。
- 周额度缺失时显示 `--%`，绝不回退到 5 小时额度。
- 右侧附着一个 **46 × 46 px** 圆形红色玻璃徽标，与球体边缘重叠约 10px。
- 徽标始终表示“可用额度重置次数”，不能表示推算轮次、已使用次数或重置倒计时。

### 3.2 悬停展开

- 指针停留 150ms 后开始展开。
- 周额度玻璃球在约 350ms 内变为 **245 × 112 px** 紫蓝玻璃胶囊。
- 原球体中心连续变为左侧紫色周额度环。
- 中间显示标题 `Codex · 本周`、本地化周重置时间和同步状态。
- 红色徽标连续滑向胶囊最右侧，仍与胶囊边缘重叠约 10px，不闪烁、不重新挂载。
- 指针离开 200ms 后收缩。

### 3.3 点击、拖动与减少动态效果

- 周版点击不翻面，也不会显示 5 小时额度。
- 点击徽标不消耗重置次数。本版本没有重置消费操作。
- 保留现有拖动阈值和原生窗口拖动；球体、胶囊和徽标都属于同一个拖动表面。
- 拖动结束后保存周版自己的窗口位置。
- 开启减少动态效果时，宽度、位置和透明度使用约 140ms 的克制过渡；不使用 3D 旋转。

## 4. 窗口几何

红色徽标是组件内部 HTML/CSS 元素，不使用 Windows 任务栏 Badge API。

- 收缩态可见宽度约为 `148 + 46 - 10 = 184px`。
- 展开态可见宽度约为 `245 + 46 - 10 = 281px`。
- 原生透明画布在四周保留 12px 光晕与阴影余量，因此目标窗口大小为收缩态 **208 × 172 px**、展开态 **305 × 136 px**。
- 展开仍以周额度环中心为锚点，主要向右生长。
- 边缘钳制、跨显示器、DPI 恢复与位置持久化必须使用周版的实际几何，不能复用标准版 172/269px 的宽度常量。
- 透明余量不得形成明显的大面积点击拦截区。

## 5. 红色玻璃徽标

徽标使用深红宝石半透明玻璃、细高光边、柔和红色外发光和轻微内阴影。数字使用高对比珊瑚红到近白的渐变，采用等宽数字，不能像游戏 HUD 或系统错误警报。

显示规则：

- `availableCount` 为 `0` 时显示 `0`。
- `1..99` 显示原值。
- 大于 99 显示 `99+`。
- 尚未成功读取时显示 `—`。
- 数字下方可显示极小的 `次`，但不能显示按钮样式或可点击暗示。
- 无障碍名称为 `剩余 N 次额度重置`；未知时为 `额度重置次数暂不可用`。

实时数据使用正常红色光泽；缓存数据降低光泽并配合胶囊内的 `显示上次同步数据`；未知数据不使用脉冲警示动画。

## 6. 重置次数数据源

Codex `app-server` 的 `account/rateLimits/read` 没有重置次数字段。周版增加独立只读数据源：

`GET https://chatgpt.com/backend-api/wham/rate-limit-reset-credits`

响应优先读取顶层整数 `available_count`。若该字段缺失但存在 `credits` 数组，则统计 `status == "available"` 且未过期的条目。负数、浮点数、异常字符串、非对象响应和无法解析的日期均视为无效响应，不能静默变成零。

该接口是 ChatGPT/Codex 当前使用的非公开后端接口，不属于稳定的公开 `app-server` 协议。实现必须把它隔离在独立的 `ResetCreditClient` 中，允许以后替换数据源，不得让接口变更破坏周额度本身的显示。

## 7. 认证、隐私与安全

重置次数只在周版运行时读取。标准版不能因为共享代码而额外读取凭据或发起该网络请求。

- 从用户当前 `CODEX_HOME`（默认 `~/.codex`）中的 `auth.json` 读取 `tokens.access_token` 与账户标识。
- 只向 `https://chatgpt.com` 发送该凭据；禁止重定向到其他主机。
- 使用 TLS、10 秒总超时和受限响应体大小。
- 访问令牌只在 Rust 内存中短暂存在，不传入 WebView、不写入缓存、不写入日志。
- 日志不得包含 Authorization、账户 ID、响应正文、信用条目 ID、用户 ID或完整服务器错误正文。
- 前端只接收规范化后的 `availableCount`、`fetchedAt` 与 `stale`。
- 本版本只做 GET 查询，不实现 `/consume`，避免误用或未经确认地消耗重置次数。

## 8. 刷新、缓存与错误处理

周额度继续由现有本地 `app-server` 链路提供。重置次数使用独立状态，不修改标准版已验证的 `RateLimitState` 语义。

重置次数在以下时机刷新：

- 周版启动并完成窗口恢复后；
- 托盘执行“刷新额度”；
- 窗口从隐藏状态重新显示；
- 系统恢复或会话解锁；
- 常驻期间每 5 分钟一次。

成功响应写入周版自己的本地缓存。失败处理：

- 有缓存：继续显示上次成功值，设置 `stale=true`。
- 无缓存：显示 `—`，状态为暂不可用。
- HTTP 401/403：标记登录状态需要刷新，但不清空仍有效的缓存。
- HTTP 429、5xx、超时或网络断开：使用有上限的退避，不高频重试。
- 数据源失败不能阻止周额度 `app-server` 启动、更新或恢复。

缓存只保存规范化数量、成功时间和可选的最早过期时间，不保存原始响应或认证数据。

## 9. 组件与边界

- `appVariant`：唯一负责判断 `standard` 或 `weekly` 构建模式。
- `WeeklyOrbitWidget`：只渲染周额度、悬停展开和红色徽标。
- `ResetCreditBadge`：只负责数量格式、视觉和无障碍语义。
- `ResetCreditClient`：读取本地 Codex 登录信息并调用受限的 ChatGPT 主机。
- `ResetCreditService`：刷新、退避、缓存和安全状态映射。
- `ResetCreditBridge`：只把规范化状态传给周版前端。
- `WindowController`：根据构建变体选择标准版或周版几何。

周额度视觉组件不能直接读取文件或发网络请求。重置次数服务不能修改额度，也不能控制窗口。

## 10. 验证与验收

自动化测试必须证明：

- 标准版行为与现有快照、交互和安装身份保持不变。
- 周版收缩态、展开态、无障碍树和构建产物中都不出现 `5H`、`5 小时`或五小时额度内容。
- 周版点击不会翻面，悬停和离开计时仍准确，拖动不会触发点击。
- 周额度更新不会重置悬停状态。
- 徽标正确显示 `—`、`0`、`1..99` 与 `99+`。
- 缓存状态和实时状态的文案、样式及无障碍名称正确。
- 解析器覆盖顶层计数、数组回退、过期条目、畸形 JSON、负数和异常类型。
- 认证与网络错误不会把未知值伪造为零，也不会在错误或调试输出中泄露令牌。
- 测试使用本地假 HTTP 服务和临时 `CODEX_HOME`，不得读取真实凭据或访问真实 ChatGPT 接口。
- 周版窗口在收缩与展开时保持徽标可见，并在工作区边缘正确钳制。
- 两个 Tauri 配置具有不同的产品名、主程序名和应用标识符。
- 两个 NSIS 安装包能够并存，两个进程能够同时运行并分别保存位置。

最终 Windows 人工验收覆盖 100%、125%、150% 和 200% DPI，深浅桌面壁纸，单/多显示器，拖动，置顶，托盘隐藏/恢复，离线缓存，未登录状态，以及标准版与周版同时安装运行。

交付物包括：

- 原版 `Gpt Orbit` NSIS 安装包；
- 新版 `Gpt Orbit Weekly` NSIS 安装包；
- 周版默认球体和悬停胶囊的桌面效果截图；
- 更新后的自动化测试与人工验收矩阵。
